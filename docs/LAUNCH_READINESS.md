## 1) 上线用户流程图（含边界）

```
┌────────────────────────────────────────────────────────────┐
│                        用户入口                            │
└────────────────────────────────────────────────────────────┘
        │
        ├─ 访客首次进入首页
        │    ├─ 输入昵称
        │    ├─ 选择：创建房间 / 加入房间
        │    └─ 最近房间入口
        │
        ├─ 创建房间
        │    ├─ 生成 roomId + adminToken + joinToken
        │    ├─ 进入房间（Admin）
        │    └─ 告知：邀请方式 / 数据生命周期
        │
        ├─ 加入房间（关键分支）
        │    ├─ 仅 roomId？ → 允许加入 or 提示需要邀请码
        │    ├─ room link / token 正确 → 加入（Member）
        │    └─ token 错误 / 过期 → 清晰提示 + 引导
        │
        └─ 回访房间
             ├─ Recent Rooms（本地存储 token）
             ├─ token 过期 → 提示房主更新
             └─ room 超过 TTL → 提示已过期
```

```
┌────────────────────────────────────────────────────────────┐
│                        房间内流程                          │
└────────────────────────────────────────────────────────────┘
        │
        ├─ WebSocket 连接
        │    ├─ 连接成功 → snapshot + role
        │    └─ 失败 → 未授权 / 房间不存在 / 已满 / 服务器异常
        │
        ├─ 正常协作
        │    ├─ 新增/编辑/完成/删除
        │    ├─ 优先级/过滤
        │    └─ 同步广播
        │
        ├─ 管理员能力
        │    ├─ 重命名
        │    ├─ 清理已完成
        │    └─ 重新生成邀请（踢人/换 token）
        │
        └─ 异常/边界
             ├─ 断线重连 → snapshot 覆盖
             ├─ 断线期间操作 → 明确失败或重试
             ├─ 房间被删除/过期 → 退出提示
             └─ 速率限制/房间满 → 反馈
```

## 2) 上线必做清单（按优先级）

### P0 必须

加入流程与权限一致化（roomId 与 token 心智对齐）  
协议与文档对齐（事件、参数、payload）  
部署与环境配置清晰可复现  
生命周期告知（24h 过期规则可见）  
错误提示一致（房间不存在、房间满、token 失效）

### P1 重要

权限 UI 反馈（Admin/Member）  
邀请安全控制（rotate token UI 入口）  
连接状态反馈（连接/断线/重连/离线操作）  
回访路径可靠（Recent Rooms 及 token 过期提示）  
基础审计感（谁操作了什么的最小提示）

### P2 增强

在线成员/人数  
只读分享  
操作历史 / 撤销（至少删除确认）  
简单监控/指标面板

---

## 3) 里程碑：MVP → 可上线

### 里程碑 A：可用闭环（上线最低标准）

加入流程完全可执行  
权限/错误/生命周期清晰  
协议/文档可信  
部署流程可复用

### 里程碑 B：信任与安全

邀请控制（rotate token）  
房主可恢复控制权  
权限状态可见

### 里程碑 C：协作可感知

在线人数/成员  
操作活动可见

### 里程碑 D：稳定可运营

监控/日志指标  
边界异常清晰反馈
